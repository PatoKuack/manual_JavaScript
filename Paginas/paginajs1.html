Introducción a JavaScript.

Buenas prácticas.

Los "punto y coma" no son necesarios en JavaScript, sin embargo, es bueno colocarlos para tener un código más legible.

Los comentarios son necesarios para guiar al programador a modificar o entender el programa ya sea porque se trabaja en equipo o porque se tiene que actualizar.

Comentarios e instrucciones.

Para comentar en un documento .js se utiliza el doble slash (//) para comentar una línea.
Para comentar un párrafo se utilizan slash y asteriscos: /*contenido*/
Se puede utilizar el estándar de doble asterisco para darme un formato más elegante, se coloca /** y el editor cerrará automáticamente el comentario de bloque con un */, después al dar un salto de línea se colocará un asterisco por cada línea en el comentario.

Con la instrucción alert("texto") aparecerá un cuadro de texto en el navegador con el texto especificado dentro de las comillas.
Con la instrucción console.log("texto") se colocará en la consola del navegador el texto especificado entre las comillas.
Con la instrucción variable[ número-de-carácter ] se obtiene el carácter ubicado en la posición especificada de un string contenido en la variable, ejemplo: console.log(variable[0])
Con el comando variable.length se obtiene el número de carácteres del string que contiene la variable.
Con el comando typeof dato imprimiremos en la consola del navegador el tipo de dato especificado ejemplo: console.log(typeof ['texto', 45, true])

Tipos de datos

  • Number: todos los tipos de números (enteros y decimales).
  • String: cadenas de texto.
    • Pueden declararse con comillas dobles "texto"
    • Pueden declararse con comillas simples 'texto'
    • Pueden declararse con backticks/backquotes (alt + BlopLock[096]) `texto` y pueden declararse variables dentro de éstas `texto ${variable}`.
  • Boolean: false o true.
  • Objetos: colección de datos colocados entre llaves y separados por comas, que tienen un identificador y un valor.
  • Array: agrupación de diferentes tipos de datos (valor) encerrados entre corchetes, con posiciones numéricas (identificadores) que van de 0 a n.
  • Null: valores que no existen, aunque no tenga carácterísticas de un objeto javascript toma este dato como objeto.
  • Undefined: es un tipo de dato no definido como por ejemplo una variable sin valor en ella.

Variables.

Reglas

Las siguientes son reglas para nombrar variables en JavaScript:
  • No pueden empezar con números, sólo con letras.
  • Se diferencian mayúsculas y minúsculas.
  • No se pueden utilizar carácteres especiales como 'ñ' o tildes.

Declaración.

Para declarar una variable se pueden utilizar tres tipos de variables var, let o const, después se coloca el nombre de la variable, ejemplo:
let videojuego;
tipo nombre;

Asignación.

Para asignar un valor a una variable se utiliza el signo igual seguido por el valor que se le desee asignar (number, string booleano, etc), ejemplo:
videojuego = "divertido";
variable = valor;

Declaración y asignación.

Se le puede asignar un valor a una variable al declararla, ejemplo:
let videojuego = "divertido";
tipo variable = valor;

Reasignación.

Se pueden reasignar valores a variables que ya contenían un valor previamente, ejemplo:
let videojuego = "divertido";
videojuego = 4;
Aquí reasignamos el valor de una variable de un valor de tipo string a uno numérico.

Asignación por referencia.

Los valores se asignan y reasignan siguiendo un orden estilo en cascada, ejemplo:
let videojuego = "divertido";
completados = videojuego;
videojuego = 4;
 En este caso videojuego acabará valiendo 4, pero completados valdrá "divertido" porque se declaró antes de la reasignación de videojuego.

Tipos de variables

var

var permite re declarar una variable, ejemplo:
var videojuego = "divertido;
var videojuego = 4;
Si no se le asigna un valor al momento de declarar una variable con var, se le asigna el valor undefined por default.
var también es muy flexible cuando se le manda a llamar, puede ser llamada desde cualquier parte del código si es que se había declarado antes, pero por lo mismo puede ocasionar problemas de seguridad por lo que no se debe usar en variables importantes que comprometan la integridad del código, por ejemplo se puede acceder a una variable var nombreVariable = valor desde la consola con el comando window.nombreVariable

let

let NO permite re declarar una variable, solamente re asignarles un valor, ejemplo:
var videojuego = "divertido";
videojuego = 4;
El comando let valor; reserva un espacio en la memoria RAM para una variable, si no se le asigna un valor no tendrá algún tipo de valor asignado.
let tiene más restricciones de seguridad, por ejemplo no se puede acceder a una variable var nombreVariable = valor desde la consola con el comando window.nombreVariable
