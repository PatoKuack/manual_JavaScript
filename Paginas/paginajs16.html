Asicronismo

JavaScript es síncrono, pero gracias a la ayuda del navegador puede funcionar de forma asíncrona ayudándole a JavaScript a ejecutar más de una tarea a la vez.

Asincronismo.

Call Stack.

Es la lista de todas las tareas que realiza el programa, la forma en que se mandan a llamar las variables y las funciones. Las tareas en el callstack se apilan de abajo hacia arriba y se llaman desde la última que mandamos a llamar hasta la más antigua. En la base de la pila reposa el Global Object, si una función llama a otra, coloca la función que es llamada encima de la pila.

Se ejecuta una tarea a la vez (síncronía), una vez que se van ejecutando las tareas se van retirando de la pila y al ejecutar todas las tareas se retira el Global object.

Memory Heap.

Es un espacio de memoria compartido para toda la aplicación ubicada en la RAM en donde se almacenan objetos y funciones en bloques de memoria de forma arbitraría y sin un orden, los cuales pueden ser usados múltiples veces a través de una referencia única.

API.

Una API es una herramienta que puedes usar para pasar información de una pieza de software a otra, puedes acceder a datos o enviarlos desde tu sitio o tu app, por ejemplo, puedes usar la API Facebook para enviarle el comportamiento de un usuario en tu e-commers y así Facebook te ayude a filtrar usuarios para enviarles publicidad especifica.

Web APIs.

Existen API que nos ayudan a manipular el DOM, a manipular el tiempo de respuesta que tienen algunas funciones y a generar peticiones. Éstas API se encuentran en el navegador, con la palabra reservada window o this se pueden visualizar las funciones que ofrece el navegador.

Callback Queue.

El código de JavaScript manda las funciones que no le competen hacia al navegador y se apilan en el Callback Queue para irse ejecutando una por una y mientras se ejecutan estas funciones, el código JavaScript ejecuta sus propias tareas. Entonces el Callback Queue es como un área de espera donde el navegador coloca sus tareas pendientes.

Event Loop.

Es un watcher u observador que mueve las tareas del Callback Queue (lado del navegador) hacia el Call Stack (lado de JavaScript) una vez que el Callback Queue está vacío para que sus tareas sean ejecutadas.

Garbage Collection.

Limpia la memoria de los datos no utilizados para no sobrecargarla y seguir trabajando sin problemas.

El algoritmo Mark & Sweep hace referencia a cuando un tipo de dato se vuelve inalcanzable para el programa.

Se toma una raíz la cual es el Objeto Global y a medida que el programa avanza, los objetos van creando o borrando referencias a sus raíces.

Cuando un objeto se queda con 0 referencias, se dice que el objeto es inalcanzable y es aquí cuando el Garbage Collector entra a hacer su trabajo (Liberar el espacio que usaban esas variables u objetos).

Desde 2012 los navegadores usan un Garbage Collector basado en este algoritmo, que ha ido recibiendo mejoras en su implementación constantemente.

Stack Overflow.

Es cuando tenemos más de las tareas de las que podemos procesar en el Call Stack.

Se genera cuando el Call Stack se llena completamente (pila de tareas). Esto pasa cuando se genera o se trabaja con bucles infinitos, recursividad y funciones. Entonces este entra en stack overflow.

function overflow(){
overflow();
}
overflow();
