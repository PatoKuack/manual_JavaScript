Inicializar proyecto.

Se crea la carpeta que contendrá al proyecto usando el comando mkdir nombre, después se accede a esa carpeta usando el comando cd nombre y ya adentro se ejecutan los siguientes comandos:

git init
Inicializa el repositorio local del proyecto en github para tener el control de cambios y versiones en nuestro proyecto.
npm init
Crea el archivo package.json con el cual se tendrá una configuración establecida, una descripción del proyecto y ciertos valores que se necesita.

Al ejecutar este comando se mostrará en la terminal una introducción sobre el archivo y pedirá los siguientes datos:

package name
Pide el nombre del proyecto, generalmente es el nombre de la carpeta. Si se da enter sin especificar un nombre se colocará el nombre por default mostrado entre paréntesis.

*ATENCÓN, el nombre debe ser original, si existe un paquete con el mismo nombre de otro paquete subido por otro usuario en el manejador de paquetes, marcará un error 403.

versión
Pide la versión con la que se inicia el proyecto, generalmente aparece 1.0.0 entre paréntesis que si se da enter se guarda esa versión inicial, aunque se puede colocar manualmente la versión deseada.
description
Pide una descripción breve del proyecto.
entry point
Pide el punto de acceso al proyecto, se puede dar enter para dejar el archivo que se sugiere entre paréntesis o se coloca la ruta o nombre del archivo que se desee (ejemplo: src/index.js).
test command
Pide comando para gestionar los test o scripts que serán importantes al correr comandos específicos a la hora de trabajar en el proyecto. Se puede dar enter sin especificar alguno y configurar esta parte más adelante.
git repository
Pide el repositorio de github u otro servicio. Se puede dar enter sin especificar alguno y especificarlo más adelante
keywords
Pide las palabras claves del proyecto, las palabras que más se cree que se utilizarán en el proyecto.
author
Pide el nombre del autor del proyecto y el correo electrónico entre signos de menor y mayor que <correo@mail.com>
license
Pide la licencia que tendrá el proyecto, como la del MIT que permite trabajar con proyectos opensource.

Una vez que se termina de ingresar los datos, se muestra una previsualización del archivo package.json. Si todo está correcto con el archivo se da enter o se escribe la palabra yes

El archivo package.json es el documento principal donde se especifica toda la configuración y los recursos informativos del proyecto.

npm set init.author.name "Fulanito"
Añade en la configuración de npm el nombre del autor.
npm set init.author.email "correo@mail.com"
Añade en la configuración de npm el e-mail del autor.
npm set init.license "MIT"
Añade en la configuración de npm la licencia deseada.
npm init -y|-yes
Crea un package.json con los datos por default y los añadidos al npm.
