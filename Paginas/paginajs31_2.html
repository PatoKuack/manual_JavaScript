Instalación de dependencias.

Lo primero que se hace es crear la carpeta que contendrá el proyecto generalmente llamado src.

Se crea el archivo index.js con el comando touch index.js que es el archivo que consumirá los recursos que se han descargado.

Desde la raíz del proyecto se instalan las dependencias o recursos que se utilizarán en el proyecto.

Una dependencia de software es: un paquete, biblioteca, o conjunto de estos, necesarios para que otro paquete pueda instalarse y ejecutarse.

Al momento de agregar dependencias se crea automáticamente una carpeta en el proyecto llamada "node_modules", en esta carpeta es donde se instalan los módulos que se agregan al proyecto. Esta carpeta es importante para que el proyecto funcione, pero no debe de ser enviada a ningún repositorio por lo que se debe ignorar apenas se crea. Para ignorarla se agrega en el archivo .gitignore el texto node_modules/

npm install paquete --save-dev
Con esta instrucción se instala un paquete y con la palabra anexa "--save" se indica que el paquete se agregará al proyecto que se subirá al servidor, si no se agrega la palabra "--save" no se subirá al servidor. La palabra anexa "-dev" indica que el paquete es necesario en el entorno de desarrollo, si no se agrega la palabra "-dev" se indica que se utilizará el paquete sólo en el entorno local.
npm i paquete -S-D
Es una forma abreviada de pedir que se instale una dependencia a npm y que se guarde como parte del proyecto y como una dependencia de desarrollo.
sudo npm istall -g nodemon
Permite generar un demonio que se encarga de escuchar un cambio de algún valor sin detener algún proceso que se esté ejecutando. Este paquete funciona de manera global, en todos los archivos.

Demonio: Servicio que corre en segundo plano en el sistema

npm list -g --depth 0
Muestra un listado de paquetes instalados de forma global (depth es la profundidad que va a buscar dentro de los paquetes).
npm install paquete -O
Permite instalar de forma opcional un paquete.
npm install paquete --dry-run
Simula la instalación del paquete especificado. Este flag indica que el paquete no va a ser instalado dentro del proyecto, simplemente es una simulación y nos muestra el output como si fuese instalado, después de esto nosotros decidimos si la instalamos o no.
npm install paquete -f | --force
Instala el paquete de forma forzada obligándolo a que sea desde el último recurso o versión y desde el servidor de NPM.
npm install
Vuelve a instalar todas las dependencias que están en el package.json
npm install paquete@versión
Instala el paquete con la versión especificada, esto puede ser necesario cuando tenemos que darle mantenimiento a una app con una versión anterior de algún paquete o quizás la versión actual es beta (ejemplo: npm install json-server@0.15.0).

Actualizar paquetes.

npm list
Muestra un listado de todo lo instalado, los paquetes con su jerarquía.
npm outdate
Muestra los paquetes que están desactualizados.
npm outdate --dd
"--dd" devuelve u output o salida más detallada del comando, en este caso muestra los paquetes desactualizados con información más detallada con la versión que se tiene de node y de npm, la sesión que se está utilizando y donde se encuentran ciertas configuraciones del proyecto.
npm update
Actualiza los paquetes que no están en la última versión.
npm install paquete@latest
Actualiza un paquete específico.

Eliminar paquetes.

npm uninstall paquete
Elimina un paquete de node_modules y del archivo package.json
npm uninstall paquete --no-save
Desinstala un paquete de node_modules, pero no del archivo package.json
