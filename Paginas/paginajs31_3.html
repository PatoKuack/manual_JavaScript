Versionado semántico.

En el archivo package.json en la parte de dependencias se tienen tres valores numéricos separados por un punto, éstos valores se encuentran al lado de cada paquete y se establecen dentro del versionado. Cada uno de los dígitos tiene un significado:

  • Primer dígito: son cambios mayores (versión 1, versión 3, etc.).
  • Segundo dígito: añaden funcionalidades que no representan un gran paso en el paquete como para considerarlo como una nueva versión.
  • Tercer dígito: estos son patch (parches), bug fixes o cambios menores.

Símbolos.

^
Cuando se mantiene el símbolo ^ (acento circunflejo o caret) dentro de la configuración del package.json, se garantiza que cuando se haga una actualización o un cambio, se actualizan sólo los cambios menores y los parches o bug fix de este paquete. Si se desea mantener el control sobre las actualizaciones garantizando que se queden en cierta versión lo mejor es elimina el caret (^).
~
Cuando se coloca el símbolo ~ (virgulilla o tilde) se recibirán actualizaciones o cambios que son parch o bug fixes.
<
Se reciben versiones menores a la indicada.
<=
Se reciben versiones menores o iguales a la indicada.
>
Se reciben versiones mayores a la indicada.
>=
Se reciben versiones mayores o iguales a la indicada.
 "dependencies": {
  "json-server": "0.15.1",
  "moment": "^2.26.0",
  "date-fns": "~2.14.0",
  "react": ">=16.12.0"
 }

A partir de la versión 5 de npm se crea un archivo llamado package-lock.json que permite tener configuraciones visibles del proyecto como las versiones, los paquetes y las dependencias que se encuentran dentro del proyecto, se puede compartir este documento con los demás desarrolladores y garantizar que las versiones que se están instalando sean las correctas, también sirve para que el servidor instale de manera automática todas estas dependencias cuando los proyectos estén en la nube.

Recursos:

Flags utilizados en instalación de paquetes.
