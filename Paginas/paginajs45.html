Variables de entorno.

Las variables de entorno son variables externas a un proyecto o aplicación que residen en el sistema operativo o en el contenedor de la aplicación que se está ejecutando. Se declaran usando mayúsculas y con formato SNAKE_CASE.

Estas variables son un espacio seguro donde se guardan datos sensibles del proyecto o aplicación. Como son sensibles, no se deben subir al repositorio y si se desea trabajar en el proyecto se deben pedir al líder del proyecto.

Para que webpack las reconozca y optimice se instala la siguiente dependencia:

npm install -D dotenv-webpack

Después, en la raíz del proyecto junto al README o al .gitignore, se crea un archivo llamado .env que es donde se almacenan las variables de entorno (no se sube al repositorio). De la misma manera se crea un archivo llamado .env.example que es en donde se colocan las mismas variables que en el archivo .env, pero sin asignarles un valor (se puede subir al repositorio).

// .env
API = https://.../
// .env.example
API =

En el archivo de configuración webpack.config.js se coloca la siguiente constante en las primeras lineas del código:

// webpack.config.js
const Dotenv = require('dotenv-webpack');

Y en la parte de los plugins del mismo archivo, se coloca lo siguiente:

// webpack.config.js
 plugins: [
  ... ,
   new Dotenv()
 ],

Después se modifican los archivos en donde se utilizan los valores de las variables de entorno de la siguiente manera:

const nombre = process.env.NOMBRE_VARIABLE;

Por último se compila el proyecto con webpack.
