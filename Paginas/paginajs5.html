Just in time compiler

Cuando se programa un código de JavaScript se genera una lista de tareas que harán didáctica a la página web.

El proceso de "just in time compiler" o "compilación en tiempo real" es cuando el motor de JavaScript funciona como un intérprete entre el código y la computadora transformando el código JavaScript en código máquina o bitecode.

Global Environment.

Una vez que el navegador recibe nuestro código, lo primero que hace es crear un entorno global el cual contiene tres cosas: un objeto global, una variable llamada "this" y un entorno de salida.

Global Object.

Este objeto es igual a Window.

this.

Esta variable depende mucho del contexto en el que se esté llamando, en el caso del entorno global, this es igual a este objeto global que también es igual a window.

Outer Environment.

También conocido como "ámbito léxico", es el bloque donde fue creada una función y que puede acceder a todos los demás elementos que están dentro del bloque de la función.

Execution context.

Después de que se generan los tres elementos del entorno global, se genera el contexto de ejecución que es conde se corre el código a través de un Stack de tareas que las apila una por una.

Parser.

Cuando empieza a interactuar JavaScript con el motor del navegador se realiza un parseo del documento completo para poder encontrar las llaves principales o Keywords.

AST.

Con las keywords del código se genera un "abstract syntax tree" o "árbol de sintaxis abstracta" que es una forma del código en donde las instrucciones se desglosan para mostrar sus componentes y las acciones que realiza.

Bitecode

El árbol sintáctico se interpreta y se convierte en bitecode el cual es un código muy básico que puede interpretar la máquina.

Profiler.

Si al interpretar el árbol sintáctico se encuentran código que se puede optimizar, el código pasa al "profiler" o "monitor" que optimiza el código, lo compila y lo regresa como bitecode para que la máquina lo pueda entender.

Hoisting.

En este último proceso (Profiler) se pueden generar errores por no utilizar bien el Hoisting que es declarar las variables y funciones de una forma adecuada para que funcionen al mandarlas llamar.
