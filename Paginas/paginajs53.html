HTTP Status Code y API Keys.

Indican el estado del servidor. Eh aquí un ejemplo con imágenes.

  • 1XX Respuestas Afirmativas
  • 2XX Respuestas satisfactorias
  • 3XX Re-direcciones
  • 4XX Error del cliente
  • 5XX Error de servidor

Ejemplo mostrando el estado de un error y su descripción:

apiURL = "https://api.com";
async function cardYuGiOh() {
 const res = await fetch(apiURL);
 const data = await res.json();
 if(res.status !== 200) {
  document.body.innerHTML = "<p>Error:" + res.status + data.message "</p>"
 }
 }

API Keys.

Las API Keys ayudan a identificar quién realiza solicitudes a los endpoints de una API.

Autenticación.

Consiste en identificar quién es cada usuario.

Autorización.

Consiste en definir que permisos tiene cada usuario.

La manera de utilizar un APIkey es obteniendo un dígito de autenticación a través de un registro en la página de la API. Después se coloca el dígito al final de la URL de consulta de la API como si fuera un query parameter.

La API de YuGiOh! no tiene APIKey, pero un ejemplo sería:

const toonMonsters = 'https://db.ygoprodeck.com/api/v7/cardinfo.php?type=toon%20monster?api_key=c08d415f-hrj3-fjt1-urh2-heej4l3c25yu';
