Otros protocolos de consumo de APIs.

Método SendBeacon.

Este es un remplazo a fetch(). Este método no espera una respuesta del servidor, es decir, solo manda solicitudes sin esperar un mensaje de error o una redirección.

Es muy útil en analytics, como cuando se quiere saber cuántas visitas tuvo un sitio web

Protocolo GraphQL

Es un remplazo de REST. Evita tener diferentes endpoints o rutas para el consumo de las APIs.

Con esta herramienta, el cliente puede definir cuanta y qué información se recibirá del backend (a esto se le conoce como Empowered Clients).

Protocolo Web Socket.

Es un remplazo de REST. Mantiene abierto el canal de comunicación con la API para consumirla de inmediato en cuanto se le haga una consulta. Esto es muy útil en aplicaciones en tiempo real.

Web 3.0

Nos permite crear o utilizar aplicaciones descentralizadas (Dapps) y ser dueños de nuestro propio contenido.
